# Game Design Snapshot Schema

The Game Design Snapshot is a JSON document that captures all creative and technical decisions about a game. It's generated by the `arcadeforge create` questionnaire and used by the generator to produce a customized game.

## Schema

### Top-level Fields

| Field | Type | Description |
|-------|------|-------------|
| `title` | string | Game name |
| `elevatorPitch` | string | One-line description |
| `genre` | `"runner"` \| `"arena"` \| `"puzzle"` | Game genre / template selection |
| `vibe` | `"retro"` \| `"cozy"` \| `"dark"` \| `"neon"` \| `"minimal"` | Visual and tonal direction |
| `playerCount` | `"single"` \| `"2-local"` \| `"leaderboard-async"` | Player mode |
| `sessionLength` | `"short"` \| `"medium"` | Target play session duration |
| `input` | `"keyboard"` \| `"touch"` \| `"gamepad"` | Primary input method |
| `difficulty` | `"casual"` \| `"moderate"` \| `"challenging"` | Difficulty preset |
| `visualStyle` | `"geometric"` \| `"pixel"` \| `"hand-drawn"` | Art direction |
| `colorPalette` | ColorPalette | Auto-generated from vibe |
| `mechanics` | Mechanics | Genre-specific gameplay parameters |
| `scope` | `"mvp"` \| `"polished"` | Polish level |
| `customNotes` | string | Free-form creator notes |
| `createdAt` | string (ISO 8601) | Generation timestamp |

### ColorPalette

| Field | Type | Description |
|-------|------|-------------|
| `background` | string (hex) | Background color |
| `primary` | string (hex) | Main accent color |
| `secondary` | string (hex) | Secondary color |
| `accent` | string (hex) | Highlight color |
| `text` | string (hex) | Text color |

### Mechanics (Runner)

| Field | Type | Default (moderate) |
|-------|------|--------------------|
| `gravity` | number | 0.5 |
| `jumpForce` | number | -11 |
| `baseSpeed` | number | 4 |
| `speedIncrement` | number | 0.002 |
| `obstacleFrequency` | number | 0.02 |

### Mechanics (Arena)

| Field | Type | Default (moderate) |
|-------|------|--------------------|
| `playerSpeed` | number | 5 |
| `bulletSpeed` | number | 10 |
| `enemySpeed` | number | 2.5 |
| `spawnRate` | number | 0.02 |
| `arenaWidth` | number | 800 |
| `arenaHeight` | number | 600 |
| `waveDifficultyCurve` | number | 1.2 |

### Mechanics (Puzzle)

| Field | Type | Default (moderate) |
|-------|------|--------------------|
| `gridWidth` | number | 7 |
| `gridHeight` | number | 7 |
| `matchSize` | number | 3 |
| `timeLimit` | number | 120 (seconds, 0 = no limit) |
| `difficultyProgression` | number | 1.1 |

## File Location

The snapshot is saved as `game-snapshot.json` in the root of each generated game directory. It's used by:
- `arcadeforge playtest` — to provide context-aware feedback suggestions
- `arcadeforge deploy` — to pre-fill hub submission data
